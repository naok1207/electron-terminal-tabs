# タブ機能動作確認シナリオ

## 1. 基本機能テスト

### 1.1 初回起動
- [ ] アプリケーションが正常に起動する
- [ ] 初期タブ（Terminal 1）が自動的に作成される
- [ ] ターミナルが表示され、入力可能な状態になる
- [ ] タブバーが表示される

### 1.2 新規タブ作成
- [ ] +ボタンをクリックすると新しいタブが作成される
- [ ] 新しいタブの名前が「Terminal 2」となる
- [ ] 新しいタブが自動的にアクティブになる
- [ ] 新しいターミナルセッションが開始される
- [ ] フォーカスが新しいターミナルに移動する

### 1.3 タブ切り替え
- [ ] タブをクリックすると切り替わる
- [ ] アクティブタブの背景色が変わる（#1e1e1e）
- [ ] 非アクティブタブの背景色が異なる（#2d2d30）
- [ ] 各タブのターミナルセッションが独立している
- [ ] 切り替え後、ターミナルにフォーカスが設定される

### 1.4 タブ削除
- [ ] タブにマウスオーバーすると×ボタンが表示される
- [ ] ×ボタンをクリックするとタブが削除される
- [ ] 削除後、適切なタブに切り替わる
- [ ] 最後のタブは削除できない

## 2. ターミナル機能テスト

### 2.1 基本的なコマンド実行
```bash
# 各タブで実行
echo "Hello from Terminal X"
ls -la
pwd
cd /tmp && pwd
```
- [ ] コマンドが正常に実行される
- [ ] 出力が正しく表示される
- [ ] 各タブで独立した作業ディレクトリ

### 2.2 インタラクティブアプリケーション
```bash
# vimのテスト
vim test.txt
# iで挿入モード、ESCでノーマルモード、:wqで保存終了

# topコマンド
top
# qで終了
```
- [ ] vimが正常に動作する
- [ ] キー入力が正しく処理される
- [ ] topコマンドが正常に表示される

### 2.3 コピー＆ペースト
- [ ] テキスト選択でクリップボードにコピー
- [ ] 右クリックでペースト
- [ ] Cmd+V（Mac）でペースト

## 3. エッジケーステスト

### 3.1 多数のタブ
- [ ] 10個のタブを作成できる
- [ ] タブバーがスクロール可能になる
- [ ] パフォーマンスの低下がない

### 3.2 ウィンドウリサイズ
- [ ] ウィンドウをリサイズしてもターミナルが適切に調整される
- [ ] リサイズ中にエラーが発生しない
- [ ] 各タブのターミナルサイズが保持される

### 3.3 高速操作
- [ ] タブを素早く切り替えてもエラーが発生しない
- [ ] タブ作成と削除を繰り返してもメモリリークしない
- [ ] ターミナル出力が大量でも安定している

## 4. エラーケーステスト

### 4.1 タブ削除時の処理
- [ ] アクティブタブを削除しても正常に動作
- [ ] 複数タブを連続で削除してもエラーなし
- [ ] タブ削除中にリサイズしてもエラーなし

### 4.2 プロセス終了
```bash
# プロセスを強制終了
exit
```
- [ ] ターミナルに「[Process exited]」と表示される
- [ ] タブは残り、新しいコマンドは実行できない

## 5. パフォーマンステスト

### 5.1 大量出力
```bash
# 大量のテキスト出力
find / -type f 2>/dev/null | head -1000
```
- [ ] スムーズにスクロールする
- [ ] UIがフリーズしない

### 5.2 メモリ使用量
- [ ] 開発者ツールでメモリ使用量を確認
- [ ] タブ作成・削除でメモリが適切に解放される
